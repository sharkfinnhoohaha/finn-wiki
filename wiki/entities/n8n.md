# n8n

Visual, node-based workflow builder. Open source (with cloud option). The [[deep-research-strategic-arbitrage|2026 strategic briefing]] frames it as the "execution layer" while the LLM provides the "intelligence layer" — a useful split when designing Phase 3 systems.

## Why it earns a page

Two reasons:

1. **n8n pipeline architecture is a marketplace specialty.** The briefing calls out Fiverr / Upwork as hubs for "n8n pipeline architects" — implying packaged n8n + Claude integrations are an established service tier. Relevant input for [[ai-agency-niches]].
2. **It's the obvious bridge between [[claude-code]] and non-developer clients.** A client who can't operate Claude Code can operate an n8n flow that calls Claude internally. Wraps the agent in a UI clients tolerate.

## Fit for Overlook

If Finn ever productizes a service that runs *for* clients (not *on* their behalf one-time), n8n is a strong candidate for the orchestration layer. Cleaner than rolling custom infrastructure, more visible than running pure Claude Code.

Tracked in [[Finn-OS/ideas/backlog]] under productized service options.
